# 百度百科词条草稿同步仓库

本仓库用于镜像 `D:\` 根目录下当前存在的 `.md` 草稿文件，方便在版本仓库中按固定顺序查看、保存和追踪这些草稿。

`D:\` 根目录是唯一来源。仓库中的编号 `.md` 文件是同步结果，不是原始草稿。需要修改草稿内容时，应修改 `D:\` 根目录中的源文件，然后重新同步到本仓库。

## 同步范围

只同步 `D:\` 根目录下的 `.md` 文件。

不处理 `D:\` 子目录里的文件，不处理非 `.md` 文件，也不修改、移动、删除或重命名 `D:\` 源文件。

## 排序方式

同步前，先读取 `D:\` 根目录下所有 `.md` 文件，并按文件修改时间从新到旧排序。

最新修改的文件排在最上面。修改时间越旧，排序越靠后。

如果两个文件的修改时间完全相同，再按文件名稳定排序，确保每次同步结果一致。

## 仓库文件命名

为了让仓库页面从上到下的显示顺序与修改时间顺序一致，仓库里的草稿文件必须按排序结果重建为三位编号文件名：

```
001_原文件名.md
002_原文件名.md
003_原文件名.md
```

其中 `001` 表示当前最新修改的源文件，编号越大表示修改时间越旧。

## 同步规则

每次同步都以 `D:\` 根目录当前存在的 `.md` 文件为准。

- `D:\` 根目录新增的 `.md` 文件，必须加入仓库。
- `D:\` 根目录已经不存在的 `.md` 文件，必须从仓库删除。
- `D:\` 源文件内容有变化时，仓库镜像必须更新。
- 源文件内容没有变化时，不做无意义内容改动。
- 仓库中的草稿文件编号和文件名必须每次按最新排序结果重建。
- `README.md` 是说明文档，不属于草稿镜像文件。

## 同步前检查

同步前应先确认仓库没有未提交改动，并拉取远端最新版本。

如果仓库已有本地改动，必须先检查这些改动，不能直接覆盖。

## 同步后校验

提交前必须完成以下校验：

1. 仓库草稿文件数量等于 `D:\` 根目录 `.md` 文件数量。
2. 仓库草稿文件名顺序与 `D:\` 源文件修改时间从新到旧的顺序完全一致。
3. 每个仓库草稿文件的 SHA256 哈希值与对应 `D:\` 源文件一致。

校验时排除 `README.md`。

## 提交与推送

只有仓库确实发生变化时才提交并推送。

如果同步后没有任何变化，不创建空提交。

提交推送后，应确认工作区干净，并简短记录当前草稿数量、最上面的文件名、最后面的文件名和提交哈希。
